# /stop — зафиксировать, запушить, выйти (admin)

Завершаешь сессию по **админке Fastcart**. По порядку.

## 1. Разобрать сессию
По диалогу и реальным изменениям: какие шаги `ROADMAP.md` завершены/начаты/частичны; новые блокеры/решения; точный следующий шаг.

## 2. Обновить ROADMAP.md
`[x]` готовым, `[~]` начатым, новые блокеры в «Блокеры», ключевые решения в «Журнал решений».

## 3. Обновить чекпоинт в CLAUDE.md
В раздел `## 📍 Последний чекпоинт` записать человеческим языком: над чем работал, что сделано (списком, с путями файлов) и какой **следующий шаг**. Обновить «Статус» и «Прошлая сессия».

## 4. Лог сессии
Определить дату и номер (посчитать `SESSION-*` за сегодня, +1). Создать `.claude/sessions/SESSION-<ГГГГ-ММ-ДД>-<NN>.md`:
```markdown
# Сессия <ГГГГ-ММ-ДД> #<NN>
## Над чем работали
## Сделано (с путями файлов)
## В процессе / не закончено
## Решения
## Блокеры / вопросы
## Следующий шаг (для /start)
```

## 5. Git (обязательно)
Выполнить по очереди, показать вывод:
1. `npm run lint`
2. `git add .`
3. `git commit -m "checkpoint: paused on <страница/фича>"`
4. `git push origin main`
Если push упал (нет remote / нет прав / другая ветка) — сообщить пользователю точную ошибку и что нужно (например `git remote add origin <url>` или сменить ветку), коммит при этом сохранён локально.

## 6. Подтвердить
Коротко: что сохранено, хэш коммита, и одной строкой «следующий шаг». После `stop` новую работу не начинать.
