# /start — начать / продолжить (admin)

Проект — **админ-панель Fastcart**. По порядку, шаги контекста не пропускать.

## 1. Контекст
1. Прочитать `CLAUDE.md`, особенно раздел `## 📍 Последний чекпоинт` (внизу).
2. Прочитать `ROADMAP.md` (фазы, чекбоксы, блокеры).
3. Прочитать последний `.claude/sessions/SESSION-*.md` (если есть).
4. Проверить состояние репозитория: `git status` и `git log --oneline -5`.

## 2. Ориентир
- Найти **первый шаг не `[x]`** в `ROADMAP.md` (или `[~]`).
- Не брать `[!]` блокеры, пока не сняты.
- Перечитать нужные `docs/` для шага (03/07 API+RTK Query, 04/09 дизайн, 08 страница, 11 системное).
- **Design-first:** открыть макет шага в `design-export/`. Нет макета → спросить пользователя, не выдумывать.

## 3. План (≤6 строк)
Какой шаг, какие файлы, какие макеты из `design-export/`. Отметить шаг `[~]`.

## 4. Выполнить
- Один шаг до конца. Правила CLAUDE.md: RTK Query (не axios в компонентах), логика в хуках, тексты через `t()`, dark/light, без `any`, без отключения линтера, 3 состояния (loading/empty/error), адаптив + mobile, копия макета.
- Прогнать `npm run type-check` + `npm run lint`, починить.
- Новый блокер → в `ROADMAP.md` (Блокеры) и продолжать с возможного.

## 5. Передача
- Сказать, что готово. Напомнить: написать `stop`, чтобы зафиксировать (чекпоинт + git push). Лог сессии тут не писать — это `stop`.
